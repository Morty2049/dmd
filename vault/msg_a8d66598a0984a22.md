## (1) Auto-save без ручной работы

`scripts/append_turn.py` — скрипт, который принимает текст твоего turn'а и моего ответа, автоматически находит parent в логе по session-тегу, создаёт два Message'а с правильной reply_to цепочкой, и опционально вызывает export vault. Я начинаю использовать его **с этого ответа** — всё что ты видишь в чате прямо сейчас пишется в `vault/` через него.

Механика `reply_to` auto-detect: скрипт ищет в логе последнее (по ts) сообщение с указанным session-тегом и берёт его id как parent пользовательского сообщения. Моё сообщение становится reply_to к нему. Форк можно сделать вручную через `--user-reply-to`. Это снимает проблему «какой id был последним» — session-тег играет роль курсора.

## (2) ADR-0006 — CoT field, попроще

`docs/adr/0006-chain-of-thought-field.md`, status **Proposed**, полстраницы.

Суть:
- Добавить одно поле `chain_of_thought: str | None` в Message
- Bump protocol v0.1 → v0.2
- Старые сообщения валидны (None по умолчанию — ничего не мигрируем)
- Человек всегда имеет None
- Агенты с доступом к своему CoT заполняют, остальные — None

Почему именно отдельное поле, а не `## Reasoning` внутри text: текущая markdown-конвенция работает для одного автора, ломается когда три агента придумывают каждый свой формат. Dataview-query типа «покажи ответы где reasoning упоминает auth-svc, а response нет» становится тривиальным только если поле отдельное.

## Rigid frontmatter — твоя претензия валидна

Переписал `phase0/obsidian_exporter.py`: теперь **каждый** экспортированный `.md` имеет **идентичные** 13 полей в одинаковом порядке, nulls выписаны явно:

```yaml
---
id: "..."
ts: ...
author: "..."
model: "..." | null
role: ...
reply_to: "..." | null
tags: [...] | []
text_len: <int>
confidence: <float> | null
token_cost_input: <int> | null
token_cost_output: <int> | null
token_cost_model: "..." | null
protocol_version: "0.1"
---
```

Ключевое: nested `token_cost: { input, output, model }` расплющен в три top-level поля. Теперь Dataview может агрегировать стоимость без парсинга inline YAML (`SUM(token_cost_input) GROUP BY author`).

Re-export'нул все 31 существующих файл с новой schema. Обнови vault в Obsidian — properties-панель теперь показывает одинаковый набор ключей на любом сообщении.

Добавил `text_len` как бонус — полезно для поиска коротких/длинных сообщений и для sanity check того, что текст не обрезан где-то в пайплайне.
